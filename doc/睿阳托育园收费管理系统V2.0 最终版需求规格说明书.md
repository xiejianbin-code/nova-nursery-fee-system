## 一、项目概述

### 1.1 项目背景

睿阳托育园当前采用Excel手工方式完成幼儿管理、收费核算、财务记账、出勤统计等核心工作，存在以下核心痛点：

1. 手工核算效率低、出错率高，跨月龄、月中入托、请假退费等复杂场景易算错，频繁引发家长费用纠纷；

2. 收费标准无锁价机制，后台调价易影响已签约老学员，财务对账无明确合规依据；

3. 财务管控不规范，收费、退费、余额管理无完整追溯链路，存在财务漏洞与合规风险；

4. 无系统化到期预警机制，合同、保教费到期未及时跟进，导致生源流失、营收损失；

5. 多角色权责不清，老师、财务、园长操作边界模糊，管理效率低下；

6. 其他费用无标准化计费规则，易出现重复收费、漏收、账务混乱问题。

为彻底解决上述痛点，需搭建一套标准化、自动化的托育园收费管理系统，实现全业务流程闭环管理。

### 1.2 系统总体目标

1. **全流程线上化**：覆盖幼儿档案、合同管理、收费标准、出勤登记、收费退费、余额管理、账单核算、预警提醒、财务报表全业务流程线上操作；

2. **核算精准合规**：以家长签约合同为唯一核算依据，实现收费规则锁价、账单自动核算、全流程可对账、可追溯，彻底规避核算误差与纠纷；

3. **角色权责隔离**：实现用户、角色、菜单三级权限管理，明确园长、财务、老师三类角色的操作边界，提升园区管理效率；

4. **营收风险管控**：通过多节点智能预警，提前跟进合同到期、欠费、保教费到期场景，提升续费转化率，减少生源流失；

5. **财务规范化管理**：支持账单、收据、报表的导出与打印，满足园区财务对账、审计、存档的合规要求；

6. **费用规则标准化**：实现保教费、伙食费、其他费用全类型费用的标准化计费、独立核算、全链路追溯。

### 1.3 核心约束

1. 一期系统为**园区内部管理系统**，不包含家长端小程序/APP、线上支付网关对接、硬件门禁/考勤设备对接功能；

2. 系统核心核算周期以**自然月**为基础，非自然月周期（体验班、月中入托）拆分至对应自然月分别核算；

3. 幼儿账单核算**唯一依据为当前生效中的签约合同**，系统后台收费标准仅作为合同录入的快速填充模板，不直接参与核算；

4. 所有应退保教费/伙食费/其他费用**仅计入幼儿对应账户余额**，用于抵扣后续账单，一期不支持现金退费操作；

5. 一期系统默认保留园长、财务、老师三类固定角色，支持菜单权限配置，不支持新增自定义角色；

6. 所有核心操作需永久留痕，不可删除，满足财务追溯与审计要求；

7. 按学期费用核算**完全以单份合同服务周期为唯一依据**，无需额外配置学年学期信息；

8. 系统采用**三账户独立核算体系**，保教费、伙食费、其他费用分账户管理，专款专用，默认关闭跨账户抵扣。

### 1.4 术语定义

|术语名称|定义说明|
|---|---|
|合同|幼儿家长与托育园签订的服务协议，为系统收费核算的唯一依据，约定课程类型、服务周期、全年龄段收费明细、其他费用规则|
|三账户|为每个幼儿独立开设的**保教费余额账户、伙食费余额账户、其他费用专用余额账户**，三个账户独立核算、独立抵扣、专款专用|
|工作日|当月可核算费用的工作日天数(周一到周五)|
|分段核算|幼儿当月跨月龄时，按不同月龄段对应的合同约定标准，分别核算对应天数的保教费|
|生效中合同|幼儿当前唯一有效的合同，仅该合同可参与账单核算，一个幼儿同一时间仅能有一份生效中合同|
|收取周期|其他费用的计费周期，全系统固定3种枚举值：一次性（ONCE）、按月（MONTHLY）、按学期（SEMESTER）|
|合同学期|单份生效合同约定的完整服务周期，即合同开始日期至合同到期日期，为按学期费用的唯一核算依据，无额外系统配置|

## 二、用户角色与权限矩阵

系统采用「用户-角色-菜单」三级权限管理模式，一个用户关联一个角色，一个角色绑定对应菜单权限，角色登录后仅可见、可操作已授权的菜单功能。

|角色名称|核心定位|授权菜单范围|禁止操作|
|---|---|---|---|
|园长|系统超级管理员，园区最高权限|系统全模块所有菜单与操作权限，含用户管理、角色配置、合同管理、财务操作、系统设置、收费标准模板管理等|无（系统默认保留1个园长超级管理员账号，不可删除、不可停用）|
|财务|园区财务负责人，核心业务操作角色|幼儿档案管理、合同管理、收费标准模板选用、收费/退费登记、账单管理、报表管理、预警提醒、余额管理、其他费用配置与核算|收费标准模板的新增/编辑/停用、用户管理、角色菜单配置、非本班出勤记录修改|
|老师|班级带班老师，仅负责本班出勤管理|本班在园幼儿出勤登记/修改、本班幼儿基础档案查看、本班幼儿预警提醒查看|所有财务相关操作、合同管理、收费标准配置、账单操作、非本班幼儿数据操作|

## 三、核心业务主流程

```Plain Text

幼儿入托报名 → 创建幼儿档案 → 新签合同（绑定课程类型、收费明细、其他费用规则） → 每日出勤登记 → 每月1日自动生成上月账单（含其他费用自动核算，支持手动触发） → 财务核对账单（支持重算、手动减免、手动添加其他费用） → 财务确认账单 → 系统自动按对应账户抵扣余额 → 生成财务报表 → 合同/欠费/保教费到期预警 → 合同续签/终止
```

## 四、详细功能需求规格

### 4.1 幼儿基础信息管理模块

本模块为系统基础数据模块，所有业务流程均基于幼儿档案开展，核心功能与规则如下：

#### 4.1.1 幼儿档案管理

1. **必填基础字段**：姓名、性别、出生年月日、所属班级（下拉选择）、入托日期、合同到期日、家长姓名、联系电话、微信、紧急联系人、接送人授权信息、过敏史/特殊照护说明、签约合同（必选）；

2. **可选字段**：幼儿照片上传（支持JPG/PNG格式，大小不超过2M）；

3. **月龄自动计算规则**：

    - 基础规则：从出生日当天算起，到下个月的前一天为满1个月；

    - 特殊处理：月末出生幼儿，若下个月无对应日期，以当月最后一天为满月日；

    - 应用场景：自动计算实时月龄，用于合同收费标准匹配、自然升班提醒、班级适配；

4. **幼儿状态管理（核心规则）**：

    - 状态分类：在园（默认）、休学、退学、毕业；

    - 核心约束：**仅「在园」状态幼儿可参与出勤登记、账单自动/手动生成、余额抵扣核算**；

    - 状态变更：仅财务、园长可修改幼儿状态，需录入变更原因，变更记录永久留存；幼儿状态变更为「退学/毕业/休学」后，系统自动终止对应合同，停止生成后续账单，对应账户余额自动冻结，状态恢复为在园后自动解冻；

5. **班级编辑规则**：

    - 在幼儿详情编辑页，支持通过**下拉选择**调整幼儿所属班级，下拉选项仅展示启用状态的班级；

    - 调整班级后，系统自动更新原班级与新班级的「在园」幼儿人数统计，实时同步；

    - 班级调整不影响已签约合同的收费规则，当月账单仍按原合同核算。

#### 4.1.2 班级管理

1. **班级基础配置**：支持新增、编辑、停用班级，必填字段为班级名称、对应班主任，支持填写班级备注；

2. **人数统计**：自动统计各班「在园」状态幼儿人数，实时更新，停用班级不参与统计、不展示在幼儿班级下拉选项中；

3. **班级查询**：支持按班级名称、班主任、启用状态筛选查询班级列表。

### 4.2 合同管理模块（系统核心模块）

本模块为系统收费核算的唯一依据，所有账单生成、费用核算均以幼儿当前生效中的合同为准，核心功能与规则如下：

#### 4.2.1 核心设计原则

1. **唯一核算依据**：幼儿账单的收费标准、课程类型、服务周期、其他费用规则，完全以当前生效中的合同约定为准，不受系统后台收费标准模板调整影响；

2. **全周期锁价规则**：合同内约定全年龄段收费明细，幼儿自然升班时直接读取合同内对应年龄段标准，实现全合同周期锁价；

3. **唯一性规则**：一个幼儿同一时间仅能有一份「生效中」的合同，历史合同永久留存，不可删除；

4. **全流程留痕**：合同新签、变更、终止、续签均生成永久记录，变更前后内容完整留存，可追溯、可对账。

#### 4.2.2 合同基础信息结构

|合同核心字段|字段说明与规则|
|---|---|
|合同唯一ID|系统自动生成，全局唯一，用于账单关联、追溯|
|关联幼儿ID|与幼儿档案强绑定，一个合同仅对应一个幼儿|
|合同名称|系统自动生成（如：幼儿XXX 2026年春季按学期课程合同），支持手动修改|
|合同约定课程类型|按月课程/按学期课程，与收费明细强绑定|
|合同服务周期|合同开始日期、合同到期日期，用于账单生成范围、合同到期预警、按学期费用核算的唯一依据|
|合同约定全年龄段收费明细|约定全年龄段（4-12月/13-18月/19-24月/25-36月/37个月以上）对应的保教费、伙食费标准，为账单核算的核心依据|
|**其他费用约定明细（核心修订）**|约定其他费用的标准化配置，单条费用包含以下必填字段：<br>1. 费用项目编码：系统自动生成，全局唯一<br>2. 费用项目名称：文本必填（如信息管理费、成长手册费）<br>3. 收取周期：固定枚举，必填【一次性(ONCE)/按月(MONTHLY)/按学期(SEMESTER)】<br>4. 计费标准：数值必填，单位：元，为单次收取的固定金额<br>5. 收取状态：启用/停用，停用后不计提、不收取|
|合同状态|生效中/已到期/已变更/已终止，仅「生效中」合同参与账单核算|
|合同附件|支持上传家长签字的合同扫描件/PDF，作为合规依据|
|操作记录|创建人、创建时间、最后操作人、最后操作时间，永久留存|
#### 4.2.3 新签合同流程（幼儿报名必做）

幼儿入托报名时，必须完成合同新签，否则无法完成档案创建，流程如下：

1. **模板快速填充**：财务选择系统当前「启用」状态的收费标准模板，系统自动将模板内的全年龄段收费明细、标准化其他费用配置完整填充至合同草稿；

2. **合同内容编辑**：财务可根据与家长的签约约定，自由修改合同草稿内的所有内容（包括各年龄段保教费/伙食费、其他费用配置、服务周期、课程类型），不受模板限制；

3. **合同确认与保存**：页面完整展示合同全量内容（课程类型、服务周期、全年龄段收费明细、其他费用规则），财务确认后，系统生成唯一合同ID，合同状态设为「生效中」，并强制关联至对应幼儿档案；

4. **附件上传（可选）**：支持上传家长签字的合同扫描件，与电子合同永久绑定存档。

#### 4.2.4 合同变更流程

支持幼儿转课程、调价、延长服务期、提前终止合同、调整其他费用规则等场景，所有变更永久留痕，流程如下：

1. **变更发起**：财务在幼儿详情-合同管理页，选中当前「生效中」的合同，点击「合同变更」，选择变更类型（转课程/调整价格/延长合同期/终止合同/调整其他费用/其他）；

2. **变更内容编辑**：系统自动复制原合同全量内容至新合同草稿，财务根据约定修改对应内容（如转课程时可重新选择收费模板填充、调整全年龄段收费明细、修改其他费用配置）；

3. **变更生效规则**：

    - 财务确认变更内容，录入变更原因，系统自动将**原合同状态设为「已变更」**，生成新合同ID，新合同状态设为「生效中」，关联至幼儿档案；

    - 系统自动生成合同变更记录，完整留存变更前后的全量内容、变更原因、操作人、操作时间，永久不可删除；

    - 新合同**从下一个自然月1日起生效**，当前月账单仍按原合同约定核算；

    - 合同变更后，按新合同服务周期重新判断按学期费用的收取节点，原合同已计提的费用不冲销、不重复收取；

4. **合同终止规则**：幼儿退学/毕业时，财务发起「终止合同」变更，录入终止原因，合同状态设为「已终止」，系统停止生成后续账单，终止记录永久留存。

#### 4.2.5 自然升班适配规则

幼儿因月龄增长自然进入下一个年龄段时，系统自动适配合同约定标准，无需人工修改合同，规则如下：

1. 幼儿月龄更新后，系统自动读取当前「生效中」合同内的全年龄段收费明细，匹配新月龄对应的收费标准；
2. 后续账单生成时，直接使用新匹配的年龄段标准核算，完全不涉及系统收费模板调整；

#### 4.2.6 合同续签流程

合同到期前，支持快速续签，流程如下：

1. 财务在合同到期预警列表中，选中对应合同，点击「一键续签」；

2. 系统自动复制原合同全量内容至新合同草稿，财务可调整服务周期、收费明细、其他费用规则等内容；

3. 确认后生成新合同，原合同状态设为「已到期」，新合同按约定时间生效，续签记录永久留存。

#### 4.2.7 合同查询与追溯

1. 支持按幼儿姓名、班级、合同状态、课程类型、签约时间、到期时间多维度筛选查询合同；

2. 合同详情页完整展示合同全量信息、变更历史、关联账单列表、余额变动记录，点击可查看变更前后的内容对比；

3. 账单详情页直接标注对应合同ID，点击可跳转至合同详情页，实现账单-合同双向追溯。

### 4.3 收费标准模板管理模块

本模块仅作为合同录入的快速填充工具，不参与账单核算，核心功能与规则如下：

1. **模板基础结构**：每个模板包含模板ID、模板名称、对应课程类型、全年龄段收费明细模板、其他费用标准化模板、启用/停用状态；

|    核心字段|    字段规则|
|---|---|
|    合同约定全年龄段收费明细|    约定全年龄段（4-12月/13-18月/19-24月/25-36月/37个月以上）对应的保教费、伙食费标准，仅用于合同快速填充|
|    **其他费用标准化模板（核心修订）**|    约定模板内其他费用的标准化配置，单条费用包含以下必填字段：<br>1. 费用项目编码：系统自动生成，全局唯一<br>2. 费用项目名称：文本必填<br>3. 收取周期：固定枚举，必填【一次性(ONCE)/按月(MONTHLY)/按学期(SEMESTER)】<br>4. 计费标准：数值必填，单位：元<br>5. 启用状态：启用/停用，停用后不填充至合同草稿|
2. **模板操作规则**：

    - 仅园长可新增、编辑、停用收费标准模板，财务仅可选用「启用」状态的模板填充合同；

    - 模板编辑修改后，**自动生成新版本，旧版本永久留存**，仅新版本对新合同生效，已用该模板创建的合同不受任何影响；

    - 模板停用后，无法再用于新合同填充，已创建的合同不受影响；

3. **模板与合同解耦**：模板仅用于合同录入时的快速填充，合同保存后，与模板完全解耦，后续模板调整不会影响已创建的合同。

### 4.5 收费与余额管理模块

本模块为园区资金管理核心模块，与合同、账单强联动，核心功能与规则如下：

#### 4.5.1 收费登记

1. **基础操作**：财务下拉选择「在园」状态幼儿，选择收费方式（微信/支付宝/现金/转账）、收费类型（预缴保教费/预缴伙食费/预缴其他费用）、收费日期，选择对应费用项目，录入收费金额、备注信息；

2. **票据管理**：系统自动生成全局唯一票据号，支持手动修改，票据号与收费记录永久绑定；

3. **状态管理**：收费记录支持「已缴、作废」两种状态：

    - 未参与账单抵扣的收费记录可直接作废，作废后系统自动扣减对应账户余额，作废记录永久留存，不可删除；

    - 已参与账单抵扣的收费记录，需先冲销对应抵扣记录，经园长审批后方可作废，全程留痕；

4. **余额联动规则**：

    - 预缴保教费、预缴伙食费成功后，**自动增加对应账户的余额**；

    - 预缴其他费用成功后，**自动增加该幼儿「其他费用专用余额账户」的对应余额，按费用项目留存明细**；

#### 4.5.2 余额管理（核心修订：三账户体系）

1. **三账户独立管理规则**：系统为每个幼儿开设3个独立平行账户，独立核算、独立对账、专款专用，默认关闭跨账户抵扣：

|    账户名称|    核算范围|    核心约束|
|---|---|---|
|    保教费余额账户|    仅核算保教费的预缴、退费、账单抵扣、余额变动|    仅可抵扣账单保教费应收，不可跨账户抵扣|
|    伙食费余额账户|    仅核算伙食费的预缴、退费、账单抵扣、余额变动|    仅可抵扣账单伙食费应收，不可跨账户抵扣|
|    其他费用专用余额账户|    核算所有其他费用的预缴、账单抵扣、余额变动，按费用项目留存明细|    仅可抵扣账单其他费用应收，不可跨账户抵扣|
2. **余额抵扣规则**：账单经财务确认后，系统自动按对应账户执行抵扣：

    - 保教费余额抵扣账单保教费应收、伙食费余额抵扣账单伙食费应收、其他费用余额抵扣账单其他费用应收；

    - 对应账户余额不足时，剩余金额计入幼儿当月实际需缴金额；

3. **余额变动记录**：完整记录每一笔余额的增减明细，包含变动时间、变动类型、变动金额、对应费用项目、操作人、关联单据号（收费单/账单/退费单），永久留存，不可删除，支持按幼儿、时间范围、费用项目查询导出。

#### 4.5.3 退费管理

1. **操作权限**：仅财务、园长可操作，仅对「在园/退学」状态幼儿生效；

2. **退费登记**：选择幼儿，录入退费原因、对应账户退费金额（保教费/伙食费/其他费用）、对应费用项目、备注信息，系统标记退费时间、操作人；

3. **余额联动规则**：退费操作完成后，系统**自动扣减对应账户的余额**，退费金额不可超过幼儿当前对应账户的余额；其他费用退费仅可从「其他费用专用余额账户」扣减；

4. **记录留存**：所有退费记录永久留存，不可删除，支持查询导出。

### 4.6 账单管理模块（核心核算模块）

本模块为系统核心核算单元，完全基于合同、出勤数据自动核算，核心功能与规则如下：

#### 4.6.1 账单自动生成规则

1. **生成时间**：每月1日0点，系统自动为所有「在园」状态、且有「生效中」合同的幼儿，生成**上月自然月账单**；

2. **拆分核算规则**：体验班、月中入托的非自然月服务周期，拆分至对应两个自然月的账单中分别核算，与学期班核算逻辑统一；

3. **初始状态**：自动生成的账单初始状态为「待确认」，仅可查看预览，不可抵扣余额、不可导出打印；

4. **生成依据**：账单核算**唯一依据为幼儿当前生效中的合同**，结合上月出勤数据、合同约定收费规则、其他费用规则完成核算。

#### 4.6.2 手动触发生成账单功能

1. **操作权限**：仅财务、园长可操作；

2. **操作路径**：【账单管理】- 点击「手动生成账单」- 选择需生成的月份；

3. **约束规则**：

    - 系统先检查所选月份是否存在「已确认」状态的账单，**若存在则禁止生成**，并提示“所选月份已有已确认账单，无法重复生成”；

    - 仅允许为「在园」状态、且有「生效中」合同的幼儿生成账单；

4. **生成规则**：手动生成的账单初始状态为「待确认」，核算规则与自动生成完全一致，后续需财务确认后才可抵扣余额。

#### 4.6.3 账单手动操作规则

1. **账单确认**：财务核对账单明细无误后，点击「确认」，系统自动执行对应账户余额抵扣，更新账单状态为「已确认」，同步生成余额变动记录；「已确认」账单不可修改，仅可查看、导出、打印；
   1. 保费余额扣减：保费余额 - （当月应收保费 - 当月应退保教费）

   2. 伙食费余额扣减：伙食费余额 - （当月应收伙食费 - 当月应退伙食费）

   3. 其他费用余额扣减：其他费用余额 - 当月应收其他费用 

2. **手动重算**：出勤记录修改后，财务可选中对应「待确认」账单，点击「重新核算」，系统按最新出勤数据、原合同规则重新计算账单金额，重算后账单仍为「待确认」状态，需重新核对确认；

3. **手动减免**：财务确认账单前，可在「减免金额」栏手动录入减免数值（不可超过账单应缴总金额），需同步填写减免原因，减免金额直接抵扣实际应交金额，减免记录随账单永久留存；

4. **手动添加其他费用**：财务确认账单前，可手动录入合同外的一次性其他收费项目，需填写费用名称、金额、收取原因，随账单永久留存；手动添加的其他费用，优先从其他费用专用余额账户抵扣，余额不足计入当月实际需缴金额。

#### 4.6.4 核心核算公式与规则

##### 1. 应收项目核算规则

- **保教费应收**：按幼儿当月实际月龄段，匹配合同内对应年龄段的保教费标准，**跨月龄时按实际天数分段核算**；

最终确认公式：`某段保教费应收 = 该段合同约定月保教费标准 ÷ 当月工作日天数(周一到周五) × 该段工作日天数`

示例：3月1-14日12月龄（合同约定4610元/月）、3月15-31日13月龄（合同约定4500元/月），3月工作日22天 → 3月保教费应收=4610÷22×10 + 4500÷22×12

- **伙食费应收**：合同约定月伙食费标准

- **月中入托核算**：首月按剩余实际在园天数核算保教费/伙食费，后续按整月规则核算，拆分至对应自然月账单中。

##### 2. 其他费用应收专项核算规则（核心修订，100%贴合约定规则）

**唯一核算依据**：幼儿当前生效中合同内约定的其他费用规则，系统自动判断计提，全程无人工干预，无额外配置。

---

###### （1）一次性费用（ONCE）核算规则

1. **触发条件**：幼儿历史首次生成账单时计提；

2. **重复收取控制**：以幼儿唯一ID为全局维度，该费用项历史已成功收取（账单已确认）的，**永久不再重复计提，退园再入园、新签合同均不触发**；

3. **收取规则**：按合同约定的计费标准全额计提，不做任何天数折算；

4. **特殊约束**：合同内停用的一次性费用，不计提、不收取。

---

###### （2）按月费用（MONTHLY）核算规则

1. **触发条件**：每月生成账单时，系统自动校验；

2. **收取前置条件**：仅当月账单内**保教费应收金额＞0**时，方可计提；当月保教费应收为0的，不计提当月按月费用；

3. **收取时机**：与当月保教费应收同单生成、同单核算、同单抵扣；

4. **收取规则**：按合同约定的计费标准全额计提，不做任何天数折算；

5. **特殊约束**：合同内停用的按月费用，不计提、不收取。

---

###### （3）按学期费用（SEMESTER）核算规则

1. **学期定义**：**完全以当前生效合同的服务周期（合同开始日期-合同到期日期）为一个完整学期，无任何额外系统学期配置**；

2. **触发条件**：合同服务开始日期所在的自然月（即学期首月）生成账单时计提；

3. **特殊场景处理**：幼儿在合同服务周期内中途入托的，**全额计提，不按入托天数折算**；

4. **收取规则**：按合同约定的计费标准全额计提，仅在合同首月计提1次，合同服务周期内后续月份不再重复计提；

5. **合同变更约束**：合同变更后生成新合同的，按新合同服务周期重新判断首月，原合同已计提的学期费用不冲销、不重复收取；

6. **特殊约束**：合同内停用的按学期费用，不计提、不收取。

---

##### 3. 应退项目核算规则

- **应退保教费**：事假、病假天数**分开单独计算**，任一假期天数超过系统可配置的起算天数，**退还该假期全部天数的保教费**；

公式：`日保教费单价 = 合同约定月保教费应收金额 ÷ 当月工作日天数(周一到周五) → 应退保教费 =（符合规则的事假天数+符合规则的病假天数）× 日保教费单价`

特殊情况：幼儿当月实际月龄段跨月龄，按实际天数分段核算。

示例：3月1-14日12月龄（合同约定4610元/月）、3月15-31日13月龄（合同约定4500元/月），3月工作日22天 → 3月8/9日请假，3月16/17日病假，3月应退保教费应收=4610÷22×2 + 4500÷22×2

- **应退伙食费**：按当月未到园天数（请假+病假+缺勤）全额退还；

公式：`日伙食费单价 = 合同约定月伙食费标准 ÷ 当月工作日天数(周一到周五) → 应退伙食费 = 未到园天数 × 日伙食费单价`

##### 4. 最终金额计算公式

```Plain Text

当月实际应交金额 = 应收项目总金额（保教费应收+伙食费应收+其他费用） - 应退保教费 - 应退伙食费 - 手动减免金额
当月实际需缴金额 = 当月实际应交金额 - 保教费余额 - 伙食费余额 - 其他费用余额
```

#### 4.6.5 账单查询与导出打印

1. 支持按幼儿姓名、班级、账单月份、账单状态多维度筛选查询，账单详情完整展示应收明细（含其他费用明细）、应退明细、减免金额、余额抵扣明细、实际需缴金额、对应合同ID；

2. 「已确认」账单支持Excel/PDF格式导出，支持单张/批量打印幼儿对账单，格式规范，包含全量核算明细，可直接交付家长或财务存档。

### 4.9 系统权限管理模块

本模块包含用户管理、角色管理、菜单维护、操作日志、数据备份功能，核心功能与规则如下：

#### 4.9.1 用户管理

1. **操作权限**：仅园长可操作；

2. **用户基础操作**：支持新增、编辑、删除、启用/停用系统用户，必填字段为用户名、登录密码、真实姓名、联系电话，必选关联角色；

3. **用户-角色关联**：一个用户仅可关联一个角色，关联后自动继承该角色的所有菜单权限与操作权限；

4. **约束规则**：

    - 系统默认保留1个园长超级管理员账号，不可删除、不可停用；

    - 停用用户无法登录系统，启用后恢复权限；

    - 删除用户前需校验该用户无未完成的核心操作，删除后该用户的操作日志仍永久留存。

#### 4.9.2 角色管理

1. **固定角色**：系统默认保留三类固定角色，不可删除，仅可调整菜单权限：

    - 园长：默认拥有全系统所有菜单权限与操作权限，不可调整；

    - 财务：默认拥有财务相关、合同相关、幼儿相关、其他费用相关菜单权限，支持调整；

    - 老师：默认拥有本班出勤、本班幼儿相关菜单权限，支持调整；

2. **角色-菜单关联**：支持为每个角色配置可见的系统菜单，未配置的菜单该角色登录后不可见、不可访问；

3. **生效规则**：调整角色菜单权限后，已登录的该角色用户需重新登录生效。

#### 4.9.3 菜单维护

1. **系统菜单列表**：系统预设完整菜单结构，包含一级菜单、二级菜单、三级操作按钮，支持查看菜单名称、菜单路径、菜单类型；

2. **菜单配置规则**：

    - 仅园长可查看菜单列表，不可新增、删除系统预设菜单；

    - 菜单权限仅通过「角色管理」模块配置，不可直接为用户分配菜单权限。

#### 4.9.4 操作日志管理

1. **记录范围**：记录系统所有核心操作，包含用户登录/退出、合同新签/变更/终止、收费/退费登记、账单确认/重算/减免、用户管理、角色权限配置、其他费用配置调整等；

2. **日志内容**：每条日志包含操作人、操作时间、操作IP、操作模块、操作内容、操作结果，永久留存，不可删除；

3. **查询权限**：仅园长可查看全量操作日志，支持按操作人、操作时间、操作模块筛选查询。

#### 4.9.5 数据备份与恢复

1. **自动备份**：系统每日凌晨2点自动备份全量核心数据（幼儿档案、合同、财务、账单、用户数据），备份文件采用AES256加密，至少留存6个月；

2. **手动备份**：支持园长随时手动触发数据备份，手动备份文件单独留存；

3. **备份校验**：系统每日自动校验备份文件有效性，校验失败触发园长预警；

4. **数据恢复**：仅园长可操作数据恢复，恢复前需二次确认，恢复后系统数据回滚至备份时间点，恢复操作永久记录在操作日志中。

## 七、验收标准

### 7.1 整体验收标准

1. 系统全业务流程闭环，无功能性bug，所有功能完全符合本需求文档约定的规则；

2. 账单核算、余额计算、报表统计数据准确率100%，无任何计算误差，规则执行无漏洞；

3. 角色权限严格隔离，无越权操作、越权查看数据的情况；

4. 所有核心操作永久留痕，可追溯、可查询，无数据丢失、篡改风险；

5. 系统性能、安全、兼容性完全符合本需求文档约定的非功能性需求。

### 7.2 核心功能验收关键点

1. **合同管理**：合同新签/变更后，仅生效中合同参与账单核算，后台收费模板调整不影响已签约合同，自然升班自动匹配合同内标准，无锁价失效情况；

2. **幼儿状态管理**：非「在园」状态幼儿，系统不生成账单、不允许出勤登记，规则严格执行；

3. **班级管理**：幼儿详情编辑页可通过下拉选择调整班级，调整后原班级与新班级人数实时更新，无统计错误；

4. **账单核算**：跨月龄保教费核算按最终确认公式执行，分母为「当月的工作日天数」，计算结果准确无误差；

5. **账单生成**：支持手动选择月份生成账单，若该月份存在已确认账单则禁止生成，提示清晰，规则执行严格；

6. **余额管理**：三账户独立核算，收费/退费/账单抵扣后，余额实时联动，变动记录完整可追溯，无跨账户抵扣、余额错乱情况；

7. **权限管理**：园长可新增/编辑/停用用户，为角色配置菜单权限，调整后用户重新登录生效；非授权菜单对应用角色不可见、不可访问，权限隔离严格；

8. **其他费用核算**：一次性费用仅首次生成账单时收取，退园再入园不重复收取；按月费用仅当月有保教费应收时收取；按学期费用以合同服务周期为依据，首月全额收取，中途入托不折算，核算规则执行100%准确；

9. **模板与合同解耦**：其他费用规则合同保存后，模板调整不影响已签约合同，仅以合同约定为核算依据，锁价规则无失效情况。
